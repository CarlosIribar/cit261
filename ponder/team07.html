Ponder: Team Activity: Comments

Overview

This activity will have us look one more time at the Great hikes application. This week we will practice using localStorage, and get another opportunity to think about the architecture of our code by adding a comments system for the hikes.

Instructions

Complete the following assignment as a team. Designate one team member as the "main driver" and collaborate on their copy of the code. Everyone on the team should be actively engaged in writing the code and contributing to the solution. Once the solution is working, make sure that everyone on the team gets a copy of the code. Each week let someone else be the "main driver" of the coding.

Core Requirements

  1. If you don't have a working copy of the Great hikes app then you can pull down this version as a starting point.

    Review the code to refresh your memory on the structure we have used. Discuss with your group what methods and properties you might need to implement your comments. Try to organize these methods and properties according to MVC. A comment should consist of at least the name of the hike it is for, a type flag, a date, and the actual text of the comment.

    Type Flag? This is just to allow us to re-use this code. Later you might want to expand on this applications and add something like favorite parks or climbs, etc. It would be nice to re-use this module for each of those. Adding a type flag allows us to grab all comments or filter for a specific group.

  2. Build out the objects and modules decided on in step 1.

    Create a new module for our Comments code. (You have a choice here whether you want to create one file and put model, view, and controller in the one file, or to create three files, either would be fine in this case) Create a class with a constructor to act as the controller. The constructor should set a local property called type when the class is instantiated.

    Create a method in the class called showCommentsList. For now it can just return some static text or log something out to the console.

    Insert your comment controller into the hike controller. Call the showCommentsList method when the list of hikes is shown to make sure that you have things connected up correctly.

    When you are looking at a specific hike it should allow you to enter a new comment, and should show all the comments below that form.

    Write the rest of the code to implement your commenting system. For now just store your comments in an array.

  3. Store It.

    Write the model code to store and retrieve the comments to/from local storage.

Stretch Goals

  1. Filtering comments

    When you are looking at a hike details screen you should only see the comments for that hike. When you are looking at all hikes you should see all comments. Change your code to support this.

  2. Controlling comment entry

    When you are looking at a hike details screen...we should be able to enter a comment, when you are looking at all hikes you should not be able to enter a new comment. Change your code to support this.
